Types of Relationships in Salesforce

  • Lookup Relationship:
    • Real-world: A student enrolls in multiple clubs, and each club can have multiple students.
    • Salesforce: A custom object (Student) can have a lookup relationship to another custom object (Club), where each Student record can reference multiple Clubs and vice versa.
  • Master-Detail Relationship:
    • Real-world: A course has multiple assignments, and each assignment is tied to a specific course.
    • Salesforce: A Course (Master) and Assignment (Detail), where deleting the Course deletes the associated Assignments.
  • Many-to-Many Relationship:
    • Real-world: Students enrolling in multiple courses and courses having multiple students.
    • Salesforce: A junction object (Enrollment) connecting Students and Courses custom objects, allowing multiple students to enroll in multiple courses.
  • Self Relationship:
    • Real-world: A student who can refer other students within the same college.
    • Salesforce: A Student object with a lookup relationship to itself to show mentor-mentee relationships among students.
  • Hierarchical Relationship:
    • Real-world: A college organizational chart showing how different departments report to the dean.
    • Salesforce: A special lookup relationship available only for the User object to create a management hierarchy, such as Department Heads reporting to the Dean.
